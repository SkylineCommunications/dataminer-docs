# What is QAOps

> [!IMPORTANT]
> This section includes information that is only applicable to Skyline employees.

In this tutorial, you learn how to navigate the QAOps application and where to find key information.

For all tutorials, always use the "QAOps Sandbox Environment": [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be).

Expected duration: 3 minutes.

## Prerequisites

- Access to [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be).

> [!IMPORTANT]
> Please contact support.boost@skyline.be to receive a username and password for access to the Sandbox system.

## Step 1: Navigate the QAOps overview

For detailed page and app information, see [QAOps main UI](xref:QAOps_Main_UI).

1. Go to [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be).

1. Select the green application named *QAOps*.

1. In the left-side menu, select *Overview*.

1. Select *Demo Test Suite* and read its description.

1. Select *Demo Configuration* and read its description.

1. Select a server and locate its creation time.

## Step 2: Navigate the QAOps test runs

For details about what a run represents, see [QAOps test run](xref:QAOps_Test_Run).

For details about the content that is executed during runs, see [QAOps test package](xref:QAOps_Test_Package).

1. Go to [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be).

1. Select the green application named *QAOps*.

1. In the left-side menu, select *Configurations*.

1. Select *Demo Configuration*.

1. Select *Demo Test Suite*.

1. Scroll through earlier test runs and open one run.

1. Scroll through the test results.

## Step 3: Locate the configuration ID and test suite ID

For more information about configurations, see [QAOps configuration](xref:QAOps_Configuration).

1. Select the green application named *QAOps*.

1. In the left-side menu, select *Configurations*.

1. Select *Demo Configuration*.

1. Select *Demo Test Suite*.

1. Locate the *Configuration ID* and *Test Suite ID* below your selections.

## Step 4: Review a test result in detail

For details about result fields and meaning, see [QAOps test result](xref:QAOps_Test_Result).

1. Open any available test run.

1. Select a test result.

1. Review the detailed message shown at the bottom of the page.

## Step 5: Check basic run metrics

For additional lifecycle and status information, see [QAOps test run](xref:QAOps_Test_Run).

1. In the selected test run, select *Show More* in the upper-right corner.

1. Review the basic metrics for that run.
