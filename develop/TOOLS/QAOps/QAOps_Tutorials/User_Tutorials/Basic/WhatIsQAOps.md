# Getting started with the QAOps User app

> [!IMPORTANT]
> This section contains information that is only applicable to Skyline employees.

In this tutorial, you will learn how to navigate the QAOps User app and discover where to find key information.

Expected duration: 3 minutes.

## Prerequisites

- Access to [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be). This "QAOps Sandbox Environment" environment should be used for every QAOps tutorial.

> [!NOTE]
> Please contact <support.boost@skyline.be> to receive a username and password for access to the Sandbox system.

## Step 1: Navigate the QAOps overview

1. Go to [https://qaops-sandbox.skyline.be](https://qaops-sandbox.skyline.be).

1. Select the green application named *QAOps* (also known as the *QAOps User* app).

1. In the navigation pane on the left, make sure *Overview* is selected.

1. Select *Demo Test Suite* and read its description.

1. Select *Demo Configuration* and read its description.

1. Select a server and locate its creation time.

> [!TIP]
> For detailed page and app information, see [QAOps main UI](xref:QAOps_Main_UI).

## Step 2: Navigate the QAOps test runs and locate the configuration ID and test suite ID

1. Still in the QAOps User app, navigate to the *Configurations* page using the navigation pane on the left.

1. Select *Demo Configuration*.

1. Select *Demo Test Suite*.

1. Locate the *Configuration ID* and *Test Suite ID* below your selections.

1. Below this, scroll through earlier test runs and select a run.

1. Scroll through the test results for the selected run.

> [!TIP]
>
> - For details about what a run represents, see [QAOps test run](xref:QAOps_Test_Run).
> - For details about the content that is executed during runs, see [QAOps test package](xref:QAOps_Test_Package).
> - For more information about configurations, see [QAOps configuration](xref:QAOps_Configuration).

## Step 3: Review a test result in detail

1. Open any available test run.

1. Select a test result.

1. Review the detailed message shown at the bottom of the page.

> [!TIP]
> For details about result fields and their meaning, see [QAOps test result](xref:QAOps_Test_Result).

## Step 4: Check basic run metrics

1. While a test run is selected, select *Show More* in the upper-right corner of the app.

1. Review the basic metrics for that run.

> [!TIP]
> For additional lifecycle and status information, see [QAOps test run](xref:QAOps_Test_Run).
