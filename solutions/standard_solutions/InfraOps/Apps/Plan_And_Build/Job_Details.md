# Job Details

To create a job, the user can select the button “New Job” on the top right. Then, the user will need to define the Job Name and Start Time. The user can also optionally define the End Time and add a Description and Remarks.

Also available are the fields Job Type (Add, Remove or Update), Priority (Critical, High, Low or Normal) and Sub State (Archived, Documentation Completed, Documentation Montage, Draft, In Progress, On Hold, Pending Approval, Pending Kickoff, Resource Allocation and Reviewed).

![Create Job – Info section](~/solutions/images/Plan_And_Build_Create_Job_Info_Section.png)

By pressing ‘Next’ after configuring the Job Information section, the user will be able to select the locations assigned to this job. Any type or number of locations can be assigned to a job:

![Create Job - Locations section](~/solutions/images/Plan_And_Build_Create_Job_Locations_Section.png)

It’s also possible to assign Assets to a job, specifying what type of action will be conducted on that asset. The list of possible actions is:

- Audited: Asset has been reviewed for compliance or performance.

- Cleaned: Physical cleaning of the asset.

- Commissioned: Asset is tested and officially put into service.

- Connected/Disconnected: Network or power connections made or removed.

- Decommissioned: Asset is retired from service but may still be physically present.

- Dismantled: Asset is broken down into components or parts.

- Expanded: Capacity or functionality has been increased.

- Inspected: Physical or logical inspection carried out.

- Locked/Unlocked: Access restrictions are applied or removed.

- Modified: Asset has been changed in some way (e.g., configuration, physical layout).

- Newly Installed: Asset is being added for the first time.

- Patched: Software or firmware updates applied.

- Provisioned: Asset is configured and made ready for use.

- Reconfigured: Settings or parameters have been changed.

- Reinstalled: Asset was previously removed and is now being installed again.

- Relocated: Asset has been moved to a different physical or logical location.

- Removed: Asset is physically taken out.

- Repaired: Faults or issues have been fixed.

- Serviced: Routine maintenance performed.

- Tagged: Asset has been labeled or identified.

- Tested: Asset has undergone diagnostic or performance testing.

- Uninstalled: Software or firmware removed from the asset.

- Upgraded: Asset has received improved hardware or software.

- Virtualized: Asset has been converted to or deployed as a virtual instance.

![Create Job - Assets Selection section](~/solutions/images/Plan_And_Build_Create_Job_Assets_Selection_Section.png)

![Create Job - Assets Selection and Action configuration](~/solutions/images/Plan_And_Build_Create_Job_Assets_Selection_And_Action_Configuration.png)

Job details after filling in the Info, Location and Assets Used sections:

![Job details side panel](~/solutions/images/Plan_And_Build_Job_Details_Side_Panel.png)

After the user fills in the necessary details, he can then assign a user and a group to the job.

This can be done by pressing the button “Assign” on the job details side panel:

![Assign Job Wizard](~/solutions/images/Plan_And_Build_Assign_Job_Wizard.png)

This will change the Job State to Assigned. Next, the user can change the state of the job to Active.

Once Active, the user can send the job for review:

![Send Job to Review Wizard](~/solutions/images/Plan_And_Build_Send_Job_To_Review_Wizard.png)

The next and final state is Complete. Once under this state, nothing can be changed on the job.
